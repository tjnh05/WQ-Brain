## Alpha因子详情

### 表达式架构
```
-rank(ts_mean(ts_delta(ts_zscore(residualized_return_india_top500_equity, 120), 5), 120))
```

#### 算子分解
1. **ts_zscore(residualized_return_india_top500_equity, 120)**
   - 功能: 对风险调整后收益进行120天窗口的时序标准化
   - 目的: 消除长期趋势和波动性影响，聚焦相对变化

2. **ts_delta(..., 5)**
   - 功能: 计算5天变化值
   - 目的: 捕捉风险因子的短期动量变化

3. **ts_mean(..., 120)**
   - 功能: 计算120天移动平均
   - 目的: 平滑信号，提高稳定性

4. **rank(...)**
   - 功能: 横截面排序
   - 目的: 生成0-1区间的标准化权重

5. **-rank(...)**
   - 功能: 符号反转
   - 目的: 根据风险因子变化方向调整多空头寸

## 经济学逻辑解释

### 因子经济学原理
该因子捕捉了**风险调整后收益的短期变化趋势**。核心逻辑链：

1. **风险调整收益**: `residualized_return_india_top500_equity`代表了剔除市场、行业等因素后的纯风险收益
2. **标准化处理**: ts_zscore消除了时序上的均值和波动性差异
3. **变化信号**: ts_delta捕捉风险收益的5天变化，反映风险偏好的短期调整
4. **趋势确认**: ts_mean(120)确认长期趋势，避免噪声交易
5. **横截面排序**: rank()识别相对机会，做多风险收益改善股票，做空恶化股票

### 市场机制
- **多头逻辑**: 风险调整后收益改善的股票，表明风险补偿增加
- **空头逻辑**: 风险调整后收益恶化的股票，表明风险补偿减少
- **中性化**: INDUSTRY中性化控制行业风险暴露

### 区域特异性 (IND)
- **印度市场特性**: 高波动性、行业集中度高
- **INDUSTRY中性化**: 有效控制印度市场的行业风险
- **TOP500股票池**: 覆盖印度主流流动性股票

## 失败案例分析

### 失败模式1: 长窗口性能下降
- **现象**: 252天窗口Sharpe降至-0.84
- **原因**: 过长窗口导致信号滞后，无法捕捉短期变化
- **教训**: IND市场需要适中的时间窗口(120天最佳)

### 失败模式2: MARKET中性化不佳
- **现象**: MARKET中性化Sharpe仅1.03
- **原因**: IND市场行业效应强于市场效应
- **教训**: IND区域优先使用INDUSTRY中性化

### 失败模式3: 基础预处理无效
- **现象**: winsorize和ts_backfill改善有限
- **原因**: 风险因子需要标准化而非简单异常值处理
- **教训**: ts_zscore对风险因子预处理效果最佳
